\chapter{Introduction}
\label{sec:introduction}

\section[Context]{Context}

The industry shift from monolithic architectures to cloud-native microservices.
Overview of the company, the Agile methodology used, and the goal of the "Smart Travel" project.

\section[Goal]{Goal}

Developing a Proof of Concept (POC) e-commerce platform while critically evaluating modern Java frameworks (Spring Boot vs. Quarkus).


\section[Thesis structure]{Thesis structure}

The thesis is structured as follows:
\begin{itemize}
    \item \textbf{Chapter 2: Background and Technologies} - Overview of microservices, reactive programming, and DevOps practices.
    \item \textbf{Chapter 3: System Architecture} - Architectural decisions, technology stack, and design patterns used in the Smart Travel project.
    \item \textbf{Chapter 4: Application Design and Implementation} - Detailed description of the development process, including code snippets and explanations of key components.
    \item \textbf{Chapter 5: Infrastructure Automation and Cloud Deployment} - Description of the CI/CD pipeline, Jenkins configuration, and deployment on OKD.
    \item \textbf{Chapter 6: Conclusion} - Summary of findings, limitations of the study, and future work directions.
\end{itemize}
